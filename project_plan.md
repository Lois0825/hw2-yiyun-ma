# Project Title： 
AI-powered Follow-up Email Generator for Business Meetings

## Target User, Workflow, and Business Value
The target user is a business professional who attends meetings.

The workflow starts with taking meeting notes and ends with sending a follow-up email.

The system takes meeting notes as input and generates a professional follow-up email as output.

This is valuable because it saves time and ensures consistent communication.

## Problem Statement and GenAI Fit
The task is to generate follow-up emails based on meeting notes.

This task benefits from language models because it involves summarization and natural language generation.

Traditional methods such as manual writing or templates are time-consuming and less flexible.

## Planned System Design and Baseline
The system will take meeting notes as input and use an LLM to generate a follow-up email.

The output will include a summary and action items in a professional tone.

I will use prompt engineering to improve output quality by refining instructions and formatting.

I will also use evaluation design by creating a test set and measuring performance based on clarity and completeness.

The baseline is manual email writing or using fixed templates without AI.

## Evaluation Plan
Success will be measured by clarity, completeness, and tone of the generated emails.

I will use a test set of at least 5 cases, including normal, edge, and failure cases.

The system will be compared against the baseline by evaluating how well it captures key points and maintains professional tone.

## Example Inputs and Failure Cases
Example inputs include:
- Meeting about project timeline
- Budget discussion
- Short notes like "budget approved"

Failure cases include:
- Very unclear notes
- Sensitive topics requiring careful wording

## Risks and Governance
The system may fail when notes are unclear or incomplete.

It should not be fully trusted in sensitive situations.

Human review is recommended before sending emails.

## Week 6 Plan
By Week 6, I expect to have a working prototype that can generate emails and an evaluation set to test performance.
